# プロダクト要件定義書

---

## プロジェクト概要

**プロジェクト名：**

MeetUP +R

**概要説明：**

OIC（立命館大学大阪いばらきキャンパス）の学生間、特に留学生と日本人学生の交流を促進するコミュニティアプリ。
共通の趣味や言語を通じて、自然な出会いや会話のきっかけを作り出す。

**開発目的：**

-   留学生が日本人と関わる機会を増やす
-   言語学習や文化交流を促進する
-   学内コミュニティの国際的つながりを強化する
-   留学生が多いという特色をいかす

**開発期間：**

2025年10月〜2025年12月

**開発体制：**

-   フロントエンド：秋田／ゆいちゃん／ゆいゆい／みきちゃん
-   バックエンド：さめ／よーた

---

## 1. 目的・ゴール定義

**1.1 プロダクトの目的**

OIC内で、言語や趣味を軸にした“リアルなつながり”をサポートする交流アプリを提供する。

---

## 2. ターゲットユーザー

**2.1 ペルソナ**

| 属性        | 内容                                           |
| ----------- | ---------------------------------------------- |
| 名前        | ミンホ（留学生）                               |
| 年齢        | 21歳                                           |
| 学部        | 情報理工学部                                   |
| 使用シーン  | 放課後や休日に日本人学生と話すきっかけを作りたい |
| 課題／ニーズ | 日本語を練習したいが、話す友達がいない         |
| 使用動機    | 趣味や言語を通じて友達を増やしたい             |

**2.2 ユーザーストーリー**

「私はOICの留学生として、日本人学生と気軽に会話したいために、このアプリを使う。」

---

## 3. 課題と価値仮説

**3.1 現状の課題**

-   留学生と日本人学生の交流機会が少ない
-   授業やイベント以外での接点が生まれにくい
-   イベントを知ることができていない

**3.2 提供価値**

-   国・言語ベースで自然にマッチング
-   AIによるトークテーマ提示で会話のきっかけを支援
-   匿名「会いたい」ボタンを作成することで会いたいと言う心理的ハードルを下げる
-   安全性の担保

---

## 4. 機能要件

### 4.1 コア機能一覧（MVP）

| 機能名                 | 内容                                       | 優先度 |
| ---------------------- | ------------------------------------------ | ------ |
| **ログイン機能**       | OIC学生限定（学内メール認証）              | **高** |
| **プロフィール作成**   | 趣味・言語・学部・性別・出身などを登録     | **高** |
| **検索機能**           | 趣味・興味ベースで相手を探す               | **高** |
| **チャット機能**       | テキストチャット（WebSocketによるリアルタイム通信） | **高** |
| 匿名「会いたい」ボタン | 相互マッチ時のみ通知                       | 中     |
| AIテーマ提案           | 会話スタート用のトピックを複数提示         | 中     |
| イベント・ミッション提示 | 学内活動を通じた出会いを促進               | 中     |

### 4.2 +α 機能

| 機能名           | 内容                       | 優先度 |
| ---------------- | -------------------------- | ------ |
| 音声通話         | チャット機能の拡張         | 低     |
| リアルタイム翻訳 | 会話中の翻訳支援           | 低     |
| スタンプ機能     | テキスト会話の雰囲気づくり | 低     |

### 4.3 機能詳細

#### ログイン

-   学内メールアドレスによる認証
-   ユーザーネームの設定

#### プロフィール

-   趣味（好きなこと）
-   言語
    -   ネイティブ言語
    -   話せる言語
    -   学びたい言語
-   学部
-   性別
-   一言（自己紹介コメント）
-   出身（居住歴が最も長い場所）

---

## 5. 非機能要件

-   **セキュリティ**: OIC学生以外のアクセスを制限する。
-   **パフォーマンス**: リアルタイムでのチャットや検索がスムーズに行えること。
-   **可用性**: 安定したサービス提供。

## 9. 懸念点と対策

| 懸念点                     | 対策                                                       |
| -------------------------- | ---------------------------------------------------------- |
| **安全性**                 | 学内メールアドレスによる厳格な認証                         |
| **会うまでのアプローチ**   | イベントやアプリ内ミッションを通じて自然な出会いを促進     |
| **グループ交流の手段**     | 一旦保留し、個人の交流にフォーカス                         |
| **機能の多さ**             | MVP（Minimum Viable Product）に絞って開発を進める          |
| **継続利用の動機付け**     | 実績や称号（バッジ）機能、スタンプ機能などを追加検討       |
| **初期ユーザー獲得**       | OICの留学生コミュニティや関連部署と連携した広報活動を検討 |
